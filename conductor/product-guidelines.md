# Product Guidelines

## Prose Style
- **Technical & Concise:** Documentation and UI text should be direct and technical, avoiding unnecessary fluff.
- **Action-Oriented:** Use imperative verbs for labels and instructions (e.g., "Select Path", "Run Query").
- **Consistent Terminology:** Use consistent naming for tools (e.g., always "Kibana Query", not "Log Search").

## UX Principles
- **Minimal Intrusion:** Debugging tools should not interfere with the app's core functionality or performance unless explicitly activated.
- **Progressive Disclosure:** Show only the necessary information by default; allow users to drill down for more detail.
- **Fast Feedback:** UI should respond quickly, especially when browsing storage or fetching logs.

## Branding & Visuals
- **Material Design:** Follow Material 3 guidelines for consistency with the Android ecosystem.
- **Functional Color Palette:** Use standard Android system colors to differentiate between informational and error states.
- **Distinct Iconography:** Use clear, recognizable icons for each utility to allow for quick identification.
